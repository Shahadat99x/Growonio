# Next Phase Plan: H1 Hero Implementation
**Focus:** Redesign and implement the homepage hero only

## Goal
Implement the locked homepage hero direction without redesigning the full homepage yet.

## Exact Scope
- Replace the current homepage hero visual approach.
- Keep the hero as a left-content / right-visual layout.
- Left side should contain:
  - badge or label
  - main headline
  - short supporting copy
  - primary CTA
  - secondary CTA
  - compact trust chips
- Right side should contain:
  - 1 desktop dashboard image
  - 1 mobile image
  - 1 HTML/CSS automation card

## Asset Requirements
- One desktop dashboard image
- One mobile image
- One coded automation card built in HTML/CSS, not as a static image

## Motion Rules
- Keep motion subtle only.
- Desktop image stays fixed and stable.
- Only the mobile image and automation card may have light motion.
- Respect reduced-motion behavior.

## Constraints
- Do not redesign the full homepage yet.
- Do not change homepage section order beyond the hero.
- Do not add new homepage sections in H1.
- Do not move homepage layout control into the database.
- Keep the tone serious, premium, and modern-tech.

## Definition of Done
- The homepage hero matches the locked H0 direction.
- The visual tone shifts toward deep violet, indigo, graphite, and off-white.
- The hero uses the three required assets/components.
- Motion remains restrained.
- The rest of the homepage stays for later phases.
